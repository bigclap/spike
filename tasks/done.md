#### **Эпик 1: Базовая структура и настройки**

Цель: Создать "скелет" расширения, добавить проверку авторизации и интерфейс для настроек.

*   **Задача 1.1: Создание базовой структуры расширения**
    *   Создать `manifest.json`. Важно: в `permissions` сразу добавить `"http://localhost/*"` (или конкретный порт, например, `"http://localhost:11434/*"`), чтобы расширение могло отправлять запросы к локальному серверу.
    *   Создать файлы для popup-интерфейса (`popup.html`, `popup.js`, `popup.css`).
    *   Создать файл фонового сценария `background.js`.

*   **Задача 1.2: Создание интерфейса настроек и управления**
    *   В `popup.html` добавить:
        *   Поле для ввода URL API локальной нейросети (например, `http://localhost:11434/api/generate`).
        *   Поле для ввода имени модели (если API это требует, как у Ollama).
        *   Кнопку "Сохранить настройки".
        *   Кнопки "Старт" и "Стоп" для запуска автоматизации.
        *   Область для вывода статуса.
    *   В `popup.js` реализовать сохранение настроек в `chrome.storage.local`.
